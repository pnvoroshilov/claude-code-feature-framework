# UseCase

Предусловие: Создан новый проект через UI \(появилась папка и фреймворк скопирован\)

Шаги:

1. Появляется окно с вопросами:
    1. Вопрос 1 \- &quot;Это простой проект или разработка продукта?&quot;
        1. Вариант ответа 1: &quot;Simple&quot;
        2. Вариант ответа 2: &quot;Development&quot;
    2. Вопрос 2 \(только если ответ на вопрос 1 = &quot;Development&quot;\) \- &quot;Это новый проект или существующий?&quot;\.
        1. Вариант ответа 1: &quot;New&quot;
        2. Вариант ответа 2: &quot;Existing&quot;
    3. Вопрос 3 \(только если ответ на Вопрос 2 был &quot;Development&quot;\- &quot;Нужна  ли помощь в setup project?&quot;
        1. Вариант ответа 1: &quot;Yes&quot;
        2. Вариант ответа 2: &quot;No&quot;
2. Если ответ на вопрос 1 = &quot;Simple&quot;, то:
    1. Установить Тип workflow в проекте = Simple
3. Иначе:
    1. если \(ответ на вопрос 2 = &quot;New&quot; и ответ на вопрос 3 =&quot;No&quot;\)  ИЛИ \(ответ на вопрос 2 = &quot;Existing&quot;\), то:
        1. Установить Тип workflow в проекте = Development
        2. Инициировать создание локального git репозитория, если его еще нет
        3. Установить параметр проекта is\_worktree\_enabled = true
    2. Если ответ на вопрос 2 = &quot;New&quot; и ответ на вопрос 3 =&quot;Yes&quot;, то:
        1. Установить Тип workflow в проекте = Development
        2. Инициировать создание локального git репозитория, если его еще нет
        3. Установить параметр проекта is\_worktree\_enabled = false
        4. Создается задача в проекте \(типа &quot;Initial implementation&quot;\)
        5. Появляется окно в котором пользователь описывает постановку задачи словами\. Текст сохраняется в описание задачи
        6. Запускаются 2 параллельных субагента \(создать если их еще нет\):
            1. Субагент Системный архитектор: 
                1. анализирует входные требования и описание продукта
                2. создает аналитические артефакты в отдельной папке в виде четких ТЗ, usecases, diagrams, user story и тд\. С четким описанием модулей и компонентов, где и что нужно создать и для чего\. Создает четкие критерии приемки и тест план
            2. Субагент UX&#x2F;UI designer:
                1. анализирует входные требования и описание продукта
                2. Создает подробное описание визуальной части \(если применимо\) в зависмости от входных требований\. Описывает стили, как выглядят экраны, какие цвета, элементы, где находятся, где переходы и тд
        7. Старт разработки с помощью подходящих субагентов в main ветке с помощью Claude \(как сейчас\)
        8. Автотестирвоание \(UI и БЭК\) согласно тестплану и критериям приемки
        9. Завершение задачи с коммитами
        10. Установить worktree = true

Дополнительная ифнормация:

Добавляется новый параметр в проект - is_worktree_enabled
